# Architectural Blueprint

This blueprint defines the structural logic of the syllabus.

| Layer          | Function                                              | Main Folder         |
| -------------- | ----------------------------------------------------- | ------------------- |
| Foundations    | Epistemic principles, axes, and relation grammar      | `01_foundations`    |
| Ontogeny       | Early embodied learning and first contact with AI     | `02_ontogeny`       |
| Praxis         | Friction, sparring, reasoning trails, co-agency       | `03_praxis`         |
| Autonomy       | Reflective practice and sovereign decision-making     | `04_autonomy`       |
| Systemic Risks | Failure modes of passive automation                   | `05_systemic_risks` |
| Protocols      | Operational loops for constructive cognitive friction | `06_protocols`      |
| Cases          | Field trials and observed learning patterns           | `07_cases`          |
| References     | Evidence map and bibliography                         | `08_references`     |

## Reading Logic

The syllabus is not a linear textbook. It is a developmental graph.

Each node should answer four questions:

1. What human capacity is being protected?
2. What AI failure mode is being resisted?
3. What kind of friction is introduced?
4. What observable behavior proves that learning is happening?
